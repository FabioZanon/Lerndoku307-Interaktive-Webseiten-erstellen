Funktion & Verwendung von i++

Die Funktion "i++" wird in den so genannten "Schleifen" verwendet. Dabei steht "i++" für nichts anderes, als "i = i+1" In einer Schleife wird eine Bedingung geprüft. Wenn sie "true" ergibt, wird ein Codeblock ausgeführt. Anschliessend wird die Bedingung erneut geprüft, und wenn sie immer noch true ist, wird der Codeblock erneut ausgeführt. Das wird so lange wiederholt, bis die Prüfung false zurückgibt. Es gibt drei Arten von Schleifen:

  • FOR
  • WHILE
  • DO..WHILE

For-Schleife: Wenn der Code in einer bestimmten Anzahl von Durchläufen ausgeführt werden muss, verwendet man eine for-Schleife. (Gebräuchlichste Art von Schleife.) Die Bedingung in einer for-Schleife ist ein Zähler, der angibt, wie oft die Schleife durchlaufen werden soll. Hier ist die Syntax für eine for-Schleife:

Gezeigt wird hier eine Variable mit dem i-Wert von 0. Die Aufgabe dieser Schleife ist nun, sie solange zu wiederholen, bis die Zahl 4 erreicht wird. Denn i soll ja kleiner sein als 5.


While-Schleife: Wenn man nicht weiss, wie oft der Code ausgeführt werden muss, verwendet man eine while-Schleife. Die Bedingung muss hier kein Zähler sein. Der Code durchläuft die Schleife so lange, wie die Bedingung true ist.

Do..While-Schleife. Die do..while-Schleife kann man gut mit der while-Schleife vergleichen. Nur dient die Do..While-Schleife dafür, dass sie IMMER mindestens einmal ausgeführt wird, selbst wenn die Bedingung gleich zu Anfang zu false ausgewertet wird.

POST und GET Methode

POST und GET benutzt man bei Web-Formularen. Wenn ein Benutzer in einem Registrierungs-Formular auf "Senden" drückt, dann sollen die Daten an einen Server geschickt werden. Dies kann mit dem "action" Attribut im Form-Tag erreicht werden. Als Pfad wird die Datei auf dem Server angegeben, welche die Daten weiterverarbeiten kann. Deshalb ist es essentiell wichtig, jedem Input einen Namen zu geben, denn anhand des Namens können nachher die Daten mit PHP auf dem Server ausgelesen und verarbeitet werden.

Mit dem "method" Attribut wird die Methode angegeben, mit welcher die Daten an den Server übermittelt werden. Hier wird zwischen "get" und "post" unterschieden. Bei GET werden die Daten in die URL geschrieben, so ähnlich wie bei Google, bei POST sind die Daten nicht zu sehen, diese Methode ist auch die Sichere, dieses Thema wird aber später noch behandelt. Serverseitige Validierung

Die Serverseitige Validierung ist eines der wichtigsten Themen, um seine Webseite / Server zu schützen. So können zum SQL-Injections verhindert werden mit nur einer Zeile PHP-Code, welche so aussieht:

$prename = mysqli_real_escape_string($_REQUIRED['prename']);. Das $_REQUIRED ist ein Platzhalter für POST oder GET, so müssen Sie sich nicht darauf achten, was Sie verwendet haben. Diese Funktion schliesst alle Strings aus, welche wie SQL-Befehle aussehen. Um die Seite vor Cross Site Scripting (XSS) zu schützen, können Sie die Variable wie folgt definieren:

$prename = htmlspecialchars($_REQUIRED['prename']);. Diese Funktion generiert aus allen Spezialzeichen wie <> einen Zeichencode, sodass XSS-Attacken unschädlich gemacht werden.

Testprotokoll

Wenn man mit der Webseite fertig ist, dann sollte man diese noch testen. Dafür wird ein Testprotokoll erstellt, auf dem steht, von wem und wann getestet wurde. Das Testprotokoll an sich teilt man in vier Spalten auf, eine indem steht was getestet wird, in der zweiten wie getestet wird, in der dritten was erhofft wird (das erhoffte Ergebnis) und in der vierten Spalte das erhaltene Ergebnis. Falls ein Fehler auftritt, soll auch dieser beschrieben werden. Jetzt könnte noch ein extra Protokoll erstellen, wo protokolliert wird, wie der Fehler behoben wird. Die Seite sollten Sie auf folgende Dinge testen:

  • Browserkompatibilität (Wird die Webseite in allen Browsern korrekt angezeigt)
  • Responsives Design korrekt (Ist die Webseite responsiv, funktioniert das auch alles und wie sieht es wirklich auf einem Smartphone aus)
  • Funktionierende Links (funktionieren alle Links auf der Seite, gilt für Verlinkungen intern sowie Verlinkungen zu anderen Webseiten)
  • Werden die Formularfelder Validiert
  • Treten Sicherheitslücken auf

Zusätzlich kann man HIER den HTML und CSS Code validieren lassen bzw. auf Fehler untersuchen.
